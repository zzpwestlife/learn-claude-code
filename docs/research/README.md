# Research Phase

> "Research and Execution must be separated." - Agent Engineering Principle #2

This directory is dedicated to the **Research Phase** of tasks. Before starting any implementation or design, research must be conducted here to explore options, understand the problem space, and make informed decisions.

## Workflow

1.  **Define the Problem**: Clearly state what needs to be solved.
2.  **Explore Options**: List possible approaches, libraries, or patterns.
3.  **Analyze Trade-offs**: Compare pros and cons of each option.
4.  **Make a Decision**: Select the best approach and justify it.
5.  **Output**: A clear decision that feeds into the **Design** or **Plan** phase.

## Anti-Patterns

- ❌ Jumping straight to "How to implement" without asking "What to implement".
- ❌ Mixing research notes with implementation plans.
- ❌ Asking the Agent to "just figure it out" during execution.

## Structure

Create a new markdown file for each research topic: `YYYY-MM-DD-topic-name.md`.
