Claude Code 的四大隐藏装备

① Skills
把团队最佳实践打包成技能

一个目录 + 一份 SKILL.md，写清触发条件和流程。无需 slash 命令，CC 按 description 自动路由。

.claude/skills/review-mr/SKILL.md

② Hooks
工具调用前后自动跑脚本

settings.json 挂钩 PreToolUse / PostToolUse / Stop。把「每次都要记得做」变成「永远不会忘」。

例：改完 .go 自动 make lint；发完 MR 评论自动打开浏览器

③ Subagents
并行派发，不污染主上下文

主 agent 派子任务，各自独立上下文，结果汇总。调研 / 审查 / 多目录探索直接并行。

单条消息发多个 Agent 调用 = 真并行

④ CLI Tools
别用 MCP，用现有 CLI

MCP 要起 server、管生命周期、调协议。主流场景一个 CLI 就够——gh / glab / kubectl，CC Bash 直接调。

MCP 会被 CLI 替代。

按 → 逐张显示，每张说 30 秒。 ① Skills：「把你们组 code review 的标准写成文件，下次就不用重复解释了。」 ② Hooks：「永远不会忘」比「记得做」可靠一个数量级。 ③ Subagents：三个独立上下文不互相污染——这是 demo 要展示的核心。 ④ CLI：个人观点，欢迎 Q&A 挑战。

4 条最小起点

  • 把组里「反复要做的检查」写成 Skill（< 200 行）
  • 在 settings.json 加一个 PreToolUse Hook 拦危险命令
  • 下次做 review / 调研，试试 Subagent 并行
  • 把常用 CLI（gh / kubectl / 内部 cli）加进 allow list
问题
  • 组里哪个重复劳动最适合做成 Skill？
  • CI 之外还有哪些「人肉检查」值得 Hook 自动化？
  • 哪些「调研类任务」可以拆给 Subagent 并行？
