# Skills Catalog (技能索引)

这是本仓库内置的所有核心技能清单。Agent 通过解析每个技能的 `description`（即触发条件）来决定何时加载该技能。
**提示**：在日常开发中，如果你需要特定行为，请直接在对话中触发对应的关键词或描述相应场景。

---

## 1. 规划与设计 (Workflow & Planning)

| 技能名称 | 触发场景 (Use when...) | 功能一句话总结 |
| :--- | :--- | :--- |
| **`brainstorming`** | 当用户想要创建或修改功能，但具体需求和方案尚未决定时触发。 | 通过多轮追问 (Grill-Me) 明确需求，产出设计文档 (spec) 和决策日志。 |
| **`design-first`** | 需要做架构/技术方案（或变更系统行为）且需求/约束/风险尚未澄清，必须先设计再实施时触发。 | 强制“先设计再实施”的铁律，产出定级、方案对比与设计摘要。 |
| **`writing-plans`** | 已有批准的 spec/设计文档，在编写代码前需要一份循序渐进的实施计划时触发。 | 根据 spec 生成带有验证命令的原子化任务计划文件。 |

## 2. 执行与代码质量 (Execution & Code Quality)

| 技能名称 | 触发场景 (Use when...) | 功能一句话总结 |
| :--- | :--- | :--- |
| **`executing-plans`** | 已有批准的计划文档，需要按步骤逐项执行时触发。 | 加载计划、执行任务、检查代码，并生成证据块。 |
| **`subagent-driven-development`** | 在当前会话中，需要为计划中的独立任务分配子 Agent 时触发。 | 为每个任务分配全新的子 Agent，并包含两阶段 Review。 |
| **`dispatching-parallel-agents`** | 面临 2 个以上相互独立、无共享状态或顺序依赖的任务时触发。 | 并发调度多个子 Agent 处理独立问题，节省时间。 |
| **`test-driven-development`** | 发生任何行为变更（新功能、修复、重构）时触发。 | 强制 TDD 流程：写测试 -> 观察失败 (RED) -> 写代码 -> 验证通过 (GREEN)。 |
| **`systematic-debugging`** | 遇到具体的错误（报错、崩溃、退退化），在修复前必须先找出根本原因时触发。 | 强制系统性排查，生成最小复现报告和调试证据块，避免盲目尝试。 |
| **`verification-before-completion`** | 当你准备声称某个任务完成（测试通过、修复完成）时触发，特别是 commit/PR 之前。 | 强制运行真实的验证命令并附上退出码和输出作为证据。 |
| **`code-review`** | 用户提供具体的审查目标（PR 链接、Git Diff、Commit 范围）时触发。 | 作为严苛的审查员，进行代码安全、性能和质量审查，产出审查报告。 |
| **`requesting-code-review`** | 完成任务、实现主要功能或准备合并代码，需要请求 Review 时触发。 | 调度 code-reviewer 子 Agent 进行早期代码审查。 |
| **`receiving-code-review`** | 收到代码审查反馈，在盲目实施前需要验证时触发。 | 评估反馈的技术合理性，有理有据地接受或反驳，避免无效修改。 |

## 3. 工作区与收尾 (Workspace & Release)

| 技能名称 | 触发场景 (Use when...) | 功能一句话总结 |
| :--- | :--- | :--- |
| **`using-git-worktrees`** | 开始新功能开发，需要与当前工作区隔离时触发。 | 自动使用 git worktree 机制准备干净的开发环境。 |
| **`finishing-a-development-branch`** | 开发完成、测试通过，需要决定如何合并或提交时触发。 | 引导完成分支开发工作，提供合并、PR 或清理等选项。 |
| **`neat-freak`** | 会话结束、里程碑达成，用户要求整理或同步文档/记忆时触发。 | 整理 README、AGENTS.md 等知识库，确保代码与文档同步。 |

## 4. 技能治理与自我优化 (Skill Management & Learning)

| 技能名称 | 触发场景 (Use when...) | 功能一句话总结 |
| :--- | :--- | :--- |
| **`using-superpowers`** | 会话开始时触发。 | 确立如何查找和使用所有其他技能的基准准则。 |
| **`writing-skills`** | 需要创建新技能、编辑现有技能或在部署前验证技能时触发。 | 运用 TDD 思想来编写、测试并完善新的 `SKILL.md`。 |
| **`darwin-skill`** | 用户提及“优化skill”、“自动优化”、“达尔文”等关键词时触发。 | 受 Karpathy 启发，通过实验和打分自主优化 `SKILL.md` 文件。 |
| **`ielts`** | 用户手动要求进行英语强化输入时触发 (Active Feeder)。 | 提取雅思词汇、错题和规则到状态文件。 |
| **`english-immersion`** | 开启英语沉浸模式，贯穿日常会话时触发。 | 被动英语沉浸与修正，记录并批量反馈语法错误。 |

## 5. 工具箱 (Tooling Skills)

| 技能名称 | 触发场景 (Use when...) | 功能一句话总结 |
| :--- | :--- | :--- |
| **`humanizer`** | 当需要编辑或检查文本，去除其中 AI 生成的痕迹使其更自然、更像人类写作时触发。 | 基于 Wikipedia 的 “Signs of AI writing” 指南，去除 AI 常用词、被动语态、过度修饰等机器痕迹。 |
| **`lark-doc-copywriting`** | 用户要求“帮我整理排版”、“标点半角化”、“中文排版修复”时触发。 | 自动化将飞书等文档的中文标点转半角、修复空格规范并整理代码块。 |
