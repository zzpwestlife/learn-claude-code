# Review Checklist

## Correctness
- Edge cases handled and validated
- Error paths covered and explicit

## Quality
- Naming is clear and consistent
- Functions/modules single responsibility

## Architecture
- Core logic isolated from IO
- Dependency injection where needed

## Performance
- No obvious inefficiencies
- Reasonable algorithmic complexity

## Security
- Input validation at boundaries
- No sensitive data in logs

## Tests
- New behavior tested
- Regressions covered
